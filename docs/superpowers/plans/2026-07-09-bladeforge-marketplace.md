# BladeForge Marketplace Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up `aquivalabs/BladeForge`, a curated company Claude Code marketplace that auto-syncs from the public source `Xaaalera/claude-skills` via a human-approved PR, gated by an eval CI and protected `main`.

**Architecture:** BladeForge owns the heavy sync logic — a workflow pulls the public source, copies every plugin except an exclude-list plus `scripts/`, regenerates `marketplace.json`, and refreshes a single rolling PR (`sync/from-personal` → `main`). The source repo gains only a tiny trigger workflow that fires `repository_dispatch` on push. An eval-gate CI is a required check; `main` is protected (PR-only).

**Tech Stack:** GitHub Actions (YAML), bash, `jq`, `gh` CLI, GitHub `repository_dispatch`, fine-grained token / GitHub App.

## Global Constraints

- Repository: `aquivalabs/BladeForge`. Marketplace `name`: `bladeforge` (lowercase). `owner`: `aquivalabs`. Display name: BladeForge; tagline theme "Blade Runner Skills".
- Source of truth: `https://github.com/Xaaalera/claude-skills` (public). It is edited ONLY for the trigger workflow + one `.gitignore` line. Plugin content is never edited for BladeForge's sake.
- Selection is an **exclude-list**, default-include. Initial excludes: `lovecraft`, `diogenes`.
- Sync direction is one-way (source → BladeForge). BladeForge never pushes back.
- Sync PR is human-approved; nothing lands in `main` unseen.
- No Cyrillic in any tracked file (language-guard hook).
- Commits: no `Co-Authored-By` trailers.
- BladeForge-owned tooling lives under `tools/sync/` (never overwritten by sync, which only replaces `plugins/` and `scripts/`).

## File Structure

**BladeForge repo (`aquivalabs/BladeForge`):**
- `tools/sync/build.sh` — the sync builder: filter plugins, copy `scripts/`, regenerate `marketplace.json`. BladeForge-owned, never synced.
- `tools/sync/exclude.txt` — one plugin name per line; the exclude-list.
- `.github/workflows/sync.yml` — on `repository_dispatch` + `workflow_dispatch`: run `build.sh`, refresh `sync/from-personal`, open/update the PR.
- `.github/workflows/eval-gate.yml` — on `pull_request`: run the synced `scripts/eval-gate.sh` against the PR diff.
- `plugins/` — synced content (arrives on first sync run; not hand-edited).
- `scripts/` — synced from source (eval-gate scripts).
- `.claude-plugin/marketplace.json` — generated by `build.sh`.
- `README.md` — reworked, teammate-facing.
- `docs/superpowers/specs/2026-07-09-bladeforge-marketplace-design.md`, `docs/superpowers/plans/2026-07-09-bladeforge-marketplace.md` — spec + this plan (moved here in Task 1).

**Human / org-admin steps (Task 7):** create repo (if `gh` lacks rights), provision dispatch token as source-repo secret, configure branch protection.

## Task 2: Sync builder script (`tools/sync/build.sh`)

**Files:**
- Create: `tools/sync/build.sh`
- Create: `tools/sync/exclude.txt`

**Interfaces:**
- Consumes: a checkout of the source repo at `$SRC` (arg 1), the BladeForge tree as CWD.
- Produces: populated `plugins/` (minus excludes), copied `scripts/`, regenerated `.claude-plugin/marketplace.json` (`name=bladeforge`, `owner.name=aquivalabs`, one entry per synced plugin).

- [ ] **Step 1: Write the exclude-list**

`tools/sync/exclude.txt`:
```
lovecraft
diogenes
```

- [ ] **Step 2: Write `build.sh`**

```bash
#!/usr/bin/env bash
# Build BladeForge content from a source-repo checkout.
# Usage: tools/sync/build.sh <path-to-source-checkout>
set -euo pipefail
SRC="${1:?usage: build.sh <source-checkout>}"
ROOT="$(git rev-parse --show-toplevel)"
cd "$ROOT"
EXCLUDE="$ROOT/tools/sync/exclude.txt"

# 1. reset synced trees
rm -rf plugins scripts
mkdir -p plugins

# 2. copy non-excluded plugins
for dir in "$SRC"/plugins/*/; do
  name="$(basename "$dir")"
  if grep -qxF "$name" "$EXCLUDE"; then
    echo "skip (excluded): $name"; continue
  fi
  cp -R "$dir" "plugins/$name"
  echo "sync: $name"
done

# 3. copy scripts/ (needed by the eval gate)
cp -R "$SRC/scripts" scripts

# 4. regenerate marketplace.json
mkdir -p .claude-plugin
plugins_json="$(
  for dir in plugins/*/; do
    name="$(basename "$dir")"
    desc="$(jq -r '.description // ""' "$dir/.claude-plugin/plugin.json" 2>/dev/null || echo "")"
    jq -n --arg n "$name" --arg d "$desc" \
      '{name:$n, description:$d, author:{name:"aquivalabs"}, source:("./plugins/"+$n)}'
  done | jq -s '.'
)"
jq -n --argjson plugins "$plugins_json" \
  '{name:"bladeforge",
    owner:{name:"aquivalabs"},
    metadata:{description:"BladeForge — aquivalabs Claude Code skills marketplace.", version:"1.0.0"},
    plugins:$plugins}' > .claude-plugin/marketplace.json
echo "wrote .claude-plugin/marketplace.json ($(echo "$plugins_json" | jq length) plugins)"
```

- [ ] **Step 3: Make executable, run against a real source checkout**

```bash
chmod +x tools/sync/build.sh
git clone --depth 1 https://github.com/Xaaalera/claude-skills /tmp/bf-src
tools/sync/build.sh /tmp/bf-src
```
Expected: `sync:` lines for each plugin, `skip (excluded): lovecraft` and `skip (excluded): diogenes`, final `wrote .claude-plugin/marketplace.json (N plugins)`.

- [ ] **Step 4: Verify excludes and validity**

```bash

jq -e '.name=="bladeforge" and .owner.name=="aquivalabs"' .claude-plugin/marketplace.json

```
Expected: `excludes OK`, then `true`, `true`.


## Task 3: Sync workflow (`.github/workflows/sync.yml`)

**Files:**
- Create: `.github/workflows/sync.yml`

**Interfaces:**
- Consumes: `tools/sync/build.sh` (Task 2), the built-in `GITHUB_TOKEN`.
- Produces: a rolling branch `sync/from-personal` and one PR into `main`, refreshed each run. Triggered by `repository_dispatch` (type `sync-from-source`) and manual `workflow_dispatch`.

- [ ] **Step 1: Write the workflow**

```yaml
name: sync-from-source
on:
  workflow_dispatch: {}
  repository_dispatch:
    types: [sync-from-source]
permissions:
  contents: write
  pull-requests: write
concurrency:
  group: sync-from-source
  cancel-in-progress: true
jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout BladeForge
        uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - name: Checkout source
        uses: actions/checkout@v4
        with:
          repository: Xaaalera/claude-skills
          path: .bf-src
          fetch-depth: 1
      - name: Configure git
        run: |
          git config user.name  "bladeforge-sync[bot]"
          git config user.email "bladeforge-sync@users.noreply.github.com"
      - name: Build content
        run: tools/sync/build.sh "$GITHUB_WORKSPACE/.bf-src"
      - name: Refresh sync branch
        run: |
          rm -rf .bf-src
          git checkout -B sync/from-personal origin/main
          git add -A plugins scripts .claude-plugin/marketplace.json
          if git diff --cached --quiet; then
            echo "no changes to sync"; exit 0
          fi
          git commit -m "chore: sync plugins from Xaaalera/claude-skills"
          git push -f origin sync/from-personal
      - name: Open or update PR
        env: { GH_TOKEN: "${{ github.token }}" }
        run: |
          if git diff --quiet origin/main -- plugins scripts .claude-plugin 2>/dev/null; then
            echo "branch matches main; nothing to PR"; exit 0
          fi
          if ! gh pr view sync/from-personal --json number >/dev/null 2>&1; then
            gh pr create --base main --head sync/from-personal \
              --title "Sync from Xaaalera/claude-skills" \
              --body "Automated sync. Review before merge. Excludes: see tools/sync/exclude.txt."
          fi
```

- [ ] **Step 2: Commit and push**

```bash

git commit -m "feat: sync workflow — refresh sync/from-personal and PR into main"
git push
```

## Task 4: Eval-gate CI (`.github/workflows/eval-gate.yml`)

**Files:**
- Create: `.github/workflows/eval-gate.yml`

**Interfaces:**
- Consumes: `scripts/eval-gate.sh` + `scripts/validate_eval.py` (present after the first sync PR includes them).
- Produces: a required PR check `eval-gate`.

Note: `scripts/eval-gate.sh` (read from source) has no stdin in CI, so it falls back to `origin/main..HEAD` — CI must fetch `main`.

- [ ] **Step 1: Write the workflow**

```yaml
name: eval-gate
on:
  pull_request:
    branches: [main]
jobs:
  eval-gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }
      - name: Ensure origin/main ref
        run: git fetch origin main:refs/remotes/origin/main
      - uses: actions/setup-python@v5
        with: { python-version: "3.12" }
      - name: Run eval gate
        run: |
          if [ ! -f scripts/eval-gate.sh ]; then
            echo "no scripts/eval-gate.sh yet (pre-seed) — skipping"; exit 0
          fi
          chmod +x scripts/eval-gate.sh scripts/validate_eval.py
          scripts/eval-gate.sh
```

- [ ] **Step 2: Commit, push, verify the check appears**

```bash
git add .github/workflows/eval-gate.yml
git commit -m "ci: eval-gate required check on PRs"
git push
```
Run: `gh pr checks sync/from-personal -R aquivalabs/BladeForge`
Expected: an `eval-gate` check listed for the open sync PR.

- [ ] **Step 2: Write the notify workflow**

`.github/workflows/notify-bladeforge.yml`:
```yaml
name: notify-bladeforge
on:
  push:
    branches: [main]
    paths:
      - 'plugins/**'
      - '.claude-plugin/**'
      - 'scripts/**'
jobs:
  notify:
    runs-on: ubuntu-latest
    steps:
      - name: Dispatch to BladeForge
        run: |
          curl -fsS -X POST \
            -H "Authorization: Bearer ${{ secrets.BLADEFORGE_DISPATCH_TOKEN }}" \
            -H "Accept: application/vnd.github+json" \
            https://api.github.com/repos/aquivalabs/BladeForge/dispatches \
            -d '{"event_type":"sync-from-source"}'
```
